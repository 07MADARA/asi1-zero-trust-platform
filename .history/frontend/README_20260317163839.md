# 🛡️ ASI-1 Industrial Threat Shield (IIoT & Cloud Defense)

**Built for API Innovate 2026 | Tracks: Cybersecurity, IoT, Web**

An advanced, AI-driven cybersecurity command center designed to protect critical enterprise infrastructure. By bridging Information Technology (IT) and Operational Technology (OT), this platform uses the agentic reasoning of **ASI:One** to detect, analyze, and remediate cyber-physical attacks on heavy industrial machinery before physical damage occurs.

---

## 💡 The Problem: Bridging IT and OT Security
Modern industrial plants and public sector units (PSUs) are increasingly connecting heavy machinery to the internet (IIoT). While standard security tools look for network breaches, they fail to understand the *mechanical* consequences of a hack. If a malicious actor compromises a Programmable Logic Controller (PLC), they don't just steal data—they alter operational limits to physically destroy equipment (e.g., the Stuxnet paradigm). 

## 🚀 The Solution: Cyber-Physical AI Defense
The **ASI-1 Industrial Threat Shield** acts as an autonomous CISO and lead engineer. Instead of just reading firewall logs, it ingests live mechanical telemetry. 

**Deep ASI:One Integration:**
* **Agentic Reasoning:** We deploy specialized AI agents (`ThreatAnalystAgent`, `VulnScannerAgent`, `PatchAdvisorAgent`) that handle distinct security domains autonomously.
* **Mechanical Telemetry Parsing:** The ASI:One API is engineered to analyze vibration and modal frequencies across a **4-stage axial compressor**. It detects when a network intrusion attempts to artificially manipulate mechanical limits, identifying cyber-physical attacks that traditional firewalls miss.
* **Automated Remediation:** ASI:One doesn't just alert; it generates contextual, ready-to-deploy bash scripts to isolate compromised PLCs dynamically.

### 1. ⚙️ Cyber-Physical IIoT Monitor (Compressor Unit A)
* Real-time telemetry tracking of vibration limits across 4 axial stages.

* Asynchronous data fetching from a MongoDB Atlas cloud backend.

## 🏗️ Architecture & Tech Stack



**Frontend (The Command Center)**
* **React.js:** Component-based UI for real-time monitoring.
* **Recharts:** High-performance data visualization for network topologies.
* **Axios:** Asynchronous API communication.

**Backend (The Intelligence Hub)**
* **FastAPI (Python):** Lightning-fast, asynchronous REST API.
* **Motor:** Async Python driver for non-blocking database operations.
* **MongoDB Atlas:** Cloud-based NoSQL database for logging threat vectors.

**AI & Orchestration**
* **ASI:One API (Fetch.ai):** Powers the core threat intelligence, vulnerability scanning, and code generation agents using models like `asi1-extended` and `asi1-agentic`.

## 📦 Setup & Installation

### Prerequisites
* Python 3.9+
* Node.js v16+
* MongoDB Atlas Account (or local MongoDB)
* ASI:One API Key

### 1. Backend Setup
Navigate to the backend directory, install dependencies, and seed the database:
```bash
cd backend
# Create and activate virtual environment (recommended)
python -m venv venv
source venv/bin/activate  # On Windows use: .\venv\Scripts\activate

# Install requirements
pip install fastapi uvicorn pydantic openai python-dotenv python-multipart motor urllib3

# Seed the database with initial IIoT telemetry and threat data
python database/seed_data.py

# Launch the FastAPI server
uvicorn main:app --reload
